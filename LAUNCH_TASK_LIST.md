## 1. 扫描范围与结论

### 1.1 本次扫描范围

- 代码：`cmd/`、`internal/`、`modules/`
- 配置与文档：`config.example.yaml`、`README.md`、`docs/*.md`
- 交付链路：`.github/workflows/ci.yml`、`.github/workflows/release.yml`、`install-release.sh`
- 质量验证：`go test ./...`、`go test -race ./...`、`go test -cover ./...`、`go vet ./...`、`gofmt -l`

### 1.2 当前结论（是否可直接上线）

- 结论：**可受控灰度上线（内网/可信用户），暂不建议直接公网全量发布**。
- 说明：A1~A5 已落地，核心安全阻塞已明显下降；仍建议补齐 P1（监控、CI 门禁、runbook、压测）再做公网放量。
- 建议：先走灰度放量，再根据监控与回归结果转正式上线。

- 单测通过：`go test ./...`

## 2. 关键风险与阻塞项（P0）


| ID    | 阻塞项              | 证据（代码位置）                                                         | 风险说明                                                                    | 必要动作                                                         | 验收标准                              |
| ----- | ---------------- | ---------------------------------------------------------------- | ----------------------------------------------------------------------- | ------------------------------------------------------------ | --------------------------------- |
| P0-01 | 管理面无鉴权           | `internal/admin/handler.go:21-63`，`cmd/gomitm/main.go:165-175`   | `/api/captures`、`/api/captures.har`、`/api/stats` 无认证，若监听地址配置到非本地会泄露流量数据 | 为 Admin API 增加鉴权中间件（至少 Bearer Token），默认仅 loopback，支持显式开启远程访问 | 未授权请求返回 401/403；文档明确默认安全行为        |
| P0-02 | SOCKS5 仅 NO AUTH | `internal/server/server.go:437-450`                              | 任意可达客户端均可使用代理，存在滥用与合规风险                                                 | 增加 SOCKS5 用户名密码认证（RFC1929）和连接拒绝日志                            | 未配置账号时默认拒绝远程来源或仅监听本地；认证失败统计可观测    |
| P0-03 | 抓包数据未脱敏          | `internal/server/server.go:1189-1197` + `1120-1131`              | Authorization/Cookie/Set-Cookie 等敏感信息会原样进入内存和 HAR                       | 增加 header/body 脱敏策略（默认掩码 + 可配置白名单）                           | `/api/captures` 与 HAR 导出默认看不到凭据明文 |
| P0-04 | 脚本执行缺少沙箱限时       | `internal/script/engine.go:460-589`                              | goja 脚本可无限循环；且 `$httpClient` 可外联，存在 DoS 与越权访问面                          | 增加脚本执行超时/中断机制；增加脚本网络能力开关（默认禁用或限制域名）                          | 死循环脚本在阈值内被终止且不拖垮主链路               |
| P0-05 | 远程模块/脚本传输协议约束不足   | `internal/module/parser.go:521-537` | 远程 sgmodule/script 若允许明文 HTTP，存在中间人篡改风险               | 远程模块与远程脚本仅允许 `https://`         | `http://` 远程资源会被明确拒绝                     |
| P0-06 | 上线文档与实现存在偏差      | `docs/UDP_ASSOCIATE_PLAN.zh-CN.md`（仍写 drop mode）                 | 认知偏差会导致运维排障和发布验收标准错误                                                    | 更新文档为当前实现现状，补齐限制与边界                                          | README/ARCH/PLAN 三份文档描述一致         |

## 4. 上线实施规划（先规划，不实施）

## 阶段 A：安全与权限基线（已完成，2026-03-20）

- 任务 A1：Admin API 鉴权与访问控制
- 任务 A2：SOCKS5 用户认证 + 默认安全监听策略
- 任务 A3：抓包脱敏（Header + 可选 JSON Body 关键字段）
- 任务 A4：脚本执行超时、取消、失败隔离
- 任务 A5：远程模块与脚本仅允许 HTTPS

交付物：

- 安全配置项文档
- 威胁模型与默认安全基线
- 覆盖认证/脱敏/超时的自动化测试

阶段验收门槛：

- 外网未授权访问无法读取任何管理数据
- 死循环脚本无法拖垮单连接以外流量
- HAR 与 captures 默认不含凭据明文

## 阶段 B：稳定性与可观测性（预计 3-4 天）

- 任务 B1：Prometheus 指标暴露与告警基线
- 任务 B2：结构化日志（连接ID、规则命中、错误分类）
- 任务 B3：资源治理（连接上限、超时、背压）
- 任务 B4：核心链路压测与容量基线（MITM/透传/UDP）

交付物：

- 监控面板草案（QPS、P95、错误率、活跃会话）
- 压测报告（含推荐机器规格）

阶段验收门槛：

- 关键指标可观测，可配置告警阈值
- 压测期间无 goroutine/FD 异常增长

## 阶段 C：发布工程与运维文档（进行中）

- 任务 C1：CI 增加 `go test -race ./...`、`go vet ./...`、覆盖率红线（已完成）
- 任务 C2：release 资产签名/校验流程完善
- 任务 C3：`install-release.sh` 校验下载资产完整性
- 任务 C4：Runbook（安装、升级、回滚、故障定位）（已完成）
- 任务 C5：文档一致性修正（README/ARCH/UDP_PLAN）

交付物：

- 可追溯发布清单
- 安装/升级/回滚 SOP

阶段验收门槛：

- 不满足门禁无法发布
- 运维手册可让新同学独立完成部署与回滚

## 阶段 D：灰度上线与放量（预计 3-5 天）

- 任务 D1：灰度环境部署（5%-10% 流量或指定用户）
- 任务 D2：连续观察 48-72 小时（稳定性、错误、性能）
- 任务 D3：分批放量（10% -> 30% -> 60% -> 100%）
- 任务 D4：每阶段保留一键回滚能力

放量门禁：

- P95 延迟、错误率、重启次数均在阈值内
- 安全事件为 0
- 用户侧关键功能无明显回归
